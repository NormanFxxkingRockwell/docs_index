# ipc 领域地图

## 领域概述
IPC Kit提供进程间通信（IPC）和远程过程调用（RPC）能力，IPC使用Binder驱动适用于设备内的跨进程通信，RPC使用软总线驱动适用于跨设备的跨进程通信。IPC和RPC采用客户端-服务端（Client-Server）模型，每个进程拥有独立的资源和内存空间，需要使用IPC和RPC实现跨进程通信。

## 核心概念
- **进程间通信**
- **远程过程调用**
- **Proxy代理**
- **Stub服务端**
- **死亡通知**
